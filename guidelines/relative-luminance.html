Versão em MathML da definição de luminância relativa

O seguinte é uma versão em MathML da definição de luminância relativa do WCAG 2.2. Consulte Software MathML - Navegadores para informação sobre navegadores e plugins que suportam MathML, que pode ser necessário para apresentar corretamente esta informação.

luminância relativa

a luminosidade relativa de qualquer ponto num espaço de cores, normalizada para 0 para o preto mais escuro e 1 para o branco mais claro

Nota

Para o espaço de cores sRGB, a luminância relativa de uma cor é definida como:

L = 0.2126 × R + 0.7152 × G + 0.0722 × B

onde R, G e B são definidos como:

  • Se RsRGB ≤ 0.04045 então R = RsRGB / 12.92 caso contrário R = ((RsRGB + 0.055) / 1.055) ^ 2.4
  • Se GsRGB ≤ 0.04045 então G = GsRGB / 12.92 caso contrário G = ((GsRGB + 0.055) / 1.055) ^ 2.4
  • Se BsRGB ≤ 0.04045 então B = BsRGB / 12.92 caso contrário B = ((BsRGB + 0.055) / 1.055) ^ 2.4

e RsRGB, GsRGB e BsRGB são definidos como:

  • RsRGB = R8bit / 255
  • GsRGB = G8bit / 255
  • BsRGB = B8bit / 255

O símbolo "^" representa a operação de exponenciação. (Fórmula retirada de [[SRGB]].)

Nota

Antes de maio de 2021, o valor 0.04045 nesta definição era diferente (0.03928). Foi atualizado a partir de uma versão anterior da especificação e não tem impacto prático nos cálculos.

Nota

A maioria dos sistemas atuais assume codificação sRGB. A menos que seja conhecido outro espaço de cores, deve-se avaliar em sRGB. Para outros espaços de cor, ver Compreender o Critério de Sucesso 1.4.3.

Nota

Se ocorrer dithering após a entrega, usa-se o valor de cor original. Se ocorrer na origem, devem ser usados valores médios das cores.

Nota

Existem ferramentas que fazem automaticamente estes cálculos ao testar contraste e flashes.
